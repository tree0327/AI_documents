# 📖 코딩 용어 사전 (초등학생도 알기 쉬운 버전!)

코딩을 하다가 모르는 단어가 나오면 여기서 찾아보세요!

## ㄱ
- **경로 (Path)**: 컴퓨터 안에서 파일이 어디에 있는지 알려주는 주소예요. "내 문서 > 숙제 > 수학" 처럼 폴더 순서를 말해요.
- **결측치 (Missing Value)**: 데이터 표에서 비어있는 칸을 말해요. 설문지에서 대답을 안 한 빈 칸 같은 거예요.

## ㄷ
- **데이터 (Data)**: 컴퓨터가 처리할 수 있는 모든 정보예요. 숫자, 글자, 그림, 소리 모두 데이터가 될 수 있어요.
- **데이터프레임 (DataFrame)**: 엑셀 표처럼 행과 열로 이루어진 데이터 덩어리를 말해요. 판다스(Pandas)라는 도구에서 주로 써요.

## ㄹ
- **라이브러리 (Library)**: 다른 사람들이 미리 만들어둔 편리한 도구 모음이에요. "도구 상자"라고 생각하면 돼요. 우리는 이 도구 상자를 가져와서(import) 쉽게 코딩할 수 있어요.
- **리스트 (List)**: 여러 개의 데이터를 순서대로 담아두는 바구니예요. `[사과, 배, 딸기]` 처럼 대괄호 `[]`로 묶어서 표현해요.

## ㅂ
- **변수 (Variable)**: 어떤 값을 담아두는 그릇이에요. 그릇에 이름을 붙여주고(`x = 10`), 나중에 그 이름을 부르면 그릇 안의 내용물(`10`)을 꺼내 쓸 수 있어요.
- **배열 (Array)**: 리스트랑 비슷한데, 수학 계산을 더 빨리, 잘하기 위해 사용하는 데이터 바구니예요. 주로 넘파이(NumPy)라는 도구에서 써요.
- **박스 플롯 (Box Plot)**: 데이터가 어디쯤에 많이 모여있는지 '상자' 모양으로 보여주는 그림이에요. 최댓값, 최솟값, 중간값 등을 한눈에 알 수 있어요. 상자 그림이라고도 해요.

## ㅅ
- **산점도 (Scatter Plot)**: 밤하늘의 별처럼 데이터를 점으로 콕콕 찍어서 보여주는 그래프예요. 두 데이터가 서로 어떤 관계인지 알 수 있어요.
- **상관관계 (Correlation)**: 두 데이터가 서로 얼마나 친한지 보여주는 거예요. 키가 크면 몸무게도 많이 나가는 것처럼, 하나가 변할 때 다른 하나도 같이 변하는 관계를 말해요.
- **서브플롯 (Subplot)**: 그래프를 여러 개로 쪼개서 한 화면에 보여주는 거예요. 만화책의 '컷' 나누기랑 비슷해요.
- **시각화 (Visualization)**: 복잡한 숫자 데이터를 눈에 쏙 들어오게 그래프나 그림으로 보여주는 것을 말해요.
- **슬라이싱 (Slicing)**: 식빵을 얇게 썰어서(slice) 원하는 만큼 가져가는 것처럼, 데이터의 일부분을 잘라내서 가져오는 것을 말해요. `[0:3]` 처럼 시작 번호와 끝 번호를 정해줘요.

## ㅇ
- **액시스 (Axes)**: 피규어(도화지) 안에 있는 실제 그래프 하나하나를 말해요. x축과 y축을 가지고 있어요.
- **이상치 (Outlier)**: 다른 친구들과 뚝 떨어져 있는 튀는 값을 말해요. (예: 우리 반 평균 키가 150cm인데 혼자 2m인 친구)
- **인덱스 (Index)**: 리스트나 데이터 표에서 데이터의 '순서 번호'예요. 컴퓨터는 숫자를 0번부터 세기 시작해요! (첫 번째는 0번, 두 번째는 1번...)
- **인덱싱 (Indexing)**: 인덱스(번호)를 이용해서 내가 원하는 데이터를 콕 집어내는 것이에요.
- **인코딩 (Encoding)**: 컴퓨터는 글자를 이해 못해서, 글자 데이터를 숫자로 바꿔주는 것을 말해요. (예: "사과" -> 1, "배" -> 2)
- **임포트 (Import)**: 라이브러리 (도구 상자)를 내 코드로 가져오는 명령어예요. "이 도구 쓸래요!" 하고 컴퓨터에게 말하는 거예요.

## ㅈ
- **전처리 (Preprocessing)**: 요리하기 전에 재료를 씻고 다듬는 과정이에요. 데이터를 분석하기 좋게 결측치(빈 값)를 채우거나 이상한 값을 고치는 작업을 말해요.
- **주석 (Comment)**: 코드 설명이나 메모를 적어놓은 글이에요. 컴퓨터는 주석을 무시하고 실행하지 않아요. (보통 `#` 기호 뒤에 써요)

## ㅋ
- **코드 (Code)**: 컴퓨터에게 내리는 명령어를 글로 적은 거예요.

## ㅍ
- **탐색적 데이터 분석 (EDA)**: 본격적인 요리(분석)를 하기 전에, 재료(데이터)가 신선한지, 어떻게 생겼는지, 맛은 어떤지 꼼꼼하게 살펴보는 과정이에요. 그래프도 그려보고 통계도 내보면서 데이터를 이해하는 시간이에요.
- **패키지 (Package)**: 라이브러리랑 거의 같은 말이에요. 도구 상자 꾸러미예요.
- **피규어 (Figure)**: 그림이 그려지는 전체 도화지나 액자를 말해요. 이 안에 여러 개의 그래프(Axes)를 그릴 수 있어요.
- **튜플 (Tuple)**: 리스트랑 비슷한데, 한번 내용을 정하면 바꿀 수 없는 주머니예요. `(사과, 배)` 처럼 소괄호 `()`를 써요.
- **함수 (Function)**: 미리 정해진 일을 해주는 마법 상자예요. 재료(입력)를 넣으면 뚝딱 요리(출력)를 만들어줘요. 예를 들어 `print()`는 괄호 안의 내용을 화면에 보여주는 함수예요.

## ㅎ
- **행/열 (Row/Column)**: 표에서 가로줄은 '행', 세로줄은 '열'이라고 해요.
- **히스토그램 (Histogram)**: 데이터가 어디에 많이 몰려있는지 '막대 기둥' 모양으로 보여주는 그래프예요. 키 순서대로 줄을 세웠을 때, 150cm대는 몇 명, 160cm대는 몇 명인지 세어보는 것과 같아요.
- **히트맵 (Heatmap)**: 숫자의 크기를 색깔의 진하기로 보여주는 표예요. 온도가 높으면 빨간색, 낮으면 파란색으로 표시하듯, 데이터가 많거나 강하면 진한 색으로 보여줘요.
