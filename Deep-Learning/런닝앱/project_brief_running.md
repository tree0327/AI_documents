# 🏃‍♂️ 프로젝트 개요: 러닝 앱 유저 이탈 방지 전략 (Running App Churn Defense)

## 1. 프로젝트 목표 (Objective)
- **문제 정의**: 러닝 앱 사용자들의 **작심삼일(Churn)** 현상을 데이터로 분석하고, 이탈을 막을 **골든타임**을 찾아내기 위함.
- **핵심 가치**: 사용자 행동 패턴을 분석하여 **"왜 그만두는지"**를 밝혀내고, 개인화된 **AI 코칭 서비스**를 통해 지속적인 러닝 유도.

## 2. 사용 데이터 (Data)
- **데이터 소스**: 러닝 앱 사용자 로그 데이터 (`running_data.csv`, 원본 JSON).
- **데이터 규모**: 약 50,000건의 러닝 기록 중 정제된 유효 유저 532명.
- **파생 변수 (Feature Engineering - 핵심)**:
    - **Overpacing_Count**: 심박수 175bpm 이상으로 무리하게 달린 횟수.
    - **Recovery_Time**: 운동 간 휴식 시간 (너무 길어지면 이탈 신호).
    - **Routine_Monotony**: 운동 패턴의 단조로움 (지루함으로 인한 이탈 분석).
    - **Churn (이탈)**: 30일 이상 기록이 없으면 이탈로 정의.

## 3. 분석 방법론 (Methodology)
### A. 유저 세그먼테이션 (Clustering)
- **K-Means Clustering**을 활용하여 유저를 3가지 그룹(초심자, 중급자, 매니아)으로 분류하여 그룹별 다른 접근 전략 수립.

### B. 모델링 (Modeling)
- **Deep Learning (PyTorch)** 모델 활용 (정확도 약 **79.2%**).
- Random Forest, XGBoost 모델과 비교 검증 완료.
- **AI Coach**: 사용자의 최근 기록을 분석하여 **"지금 3일째 쉬고 계시네요, 오늘 3km 가볍게 어떠세요?"**와 같은 맞춤형 피드백 생성 로직 구현.

## 4. 주요 성과 및 결론 (Results)
- **이탈 원인 규명**: 초보자의 경우 **"무리한 페이스(Overpacing)"**가 이탈의 가장 큰 원인임을 데이터로 입증.
- **웹 서비스 구현**: Streamlit을 활용하여 대시보드 및 AI 예측 서비스 구축 (`app.py`).
- **서비스 가치**: 단순 기록 측정이 아닌, 사용자 이탈을 사전에 방지하는 **"러닝 페이스메이커"** 역할 수행 가능.
